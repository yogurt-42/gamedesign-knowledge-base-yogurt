# 环境叙事 (Environmental Storytelling)

## 核心概念：指数叙事 (Indexical Storytelling)

**不是告诉玩家发生了什么，而是让玩家自己发现。**

### 指示符 (Index) 示例
| 场景元素 | 暗示信息 | 玩家推理 |
|---------|---------|---------|
| 墙壁弹孔 | 之前发生过战斗 | 这里不安全 |
| 拖拽血迹 | 有人被拖走 | 附近有危险 |
| 烧焦尸体在电围栏旁 | 触电死亡 | 围栏带电 |
| 散落的文件 | 匆忙撤离 | 突发事故 |

## GDC 经典原则

### 1. 引导移动
利用光线、颜色对比、地标引导玩家自然流向目标。

### 2. 传达 affordance
场景暗示可交互性：
- 破损的墙 → 可以炸开
- 通风口尺寸 → 可以爬入
- 水坑 + 电线 → 电击危险

### 3. 强化身份
场景风格传达信息：
- 整洁的实验室 → 秩序、控制
- 杂乱的办公室 → 混乱、紧急
- 深层设施的邪教符号 → 疯狂、未知

### 4. 提供叙事语境
场景讲述"这里发生过什么"：
- 生日蛋糕 + 聚会装饰 + 血迹 = 悲剧发生时正在庆祝
- 儿童画 + 束缚带 = 不人道的实验

## 应用到SCP收容所

### 安全区叙事
```
办公区:
- 倒下的咖啡杯（事发时正在工作）
- 半开的抽屉（有人匆忙找东西）
- 墙上的员工照片（人性化受害者）
```

### 危险区叙事
```
深层收容区:
- 抓痕墙壁（某物试图逃脱）
- 干涸的血迹（多次事故）
- 前员工的最后录音（录音设备）
```

### 收容物相关的环境叙事
不是直接展示收容物，而是展示它的"痕迹"：
- K-661区域：模仿人类行为的涂鸦、倒着写的警告
- K-999区域：粘液的痕迹、被hug过的员工留下的感谢信

## 2D俯视角实现技巧

### 视觉层级
1. **前景**：可交互物体（高亮）
2. **中景**：叙事细节（血迹、弹孔）
3. **背景**：场景氛围（灯光、破损）

### 文档散落系统
- 随机散落前员工的笔记
- 用颜色/图标分类：日志（蓝）、警告（红）、个人（黄）
- 拼凑完整故事需要收集多个碎片

### 动态环境变化
- 第一次经过时：整洁
- 收容物突破后：混乱
- 玩家处理后：部分恢复但有痕迹

## 避免的错误

### ❌ BioShock 的教训
尸体旁边有自动售货机和钱：
- 暗示"贪婪导致死亡"的主题
- 但玩家行为（搜刮）与场景叙事（悲剧）冲突

### ✅ 正确做法
玩家行为应强化而非破坏叙事：
- 搜刮尸体时播放其生前录音 → 道德重量
- 拆解设备时显示曾是某人工作区 → 牺牲感
