## 机制描述九要素

### 1. Action（动作）
玩家执行的动作是什么？

**示例**：
- 按下空格键跳跃
- 点击物品栏使用物品
- 按住右键瞄准

### 2. Purpose（目的）
玩家为什么执行这个动作？

**示例**：
- 跳跃：越过障碍、到达高处
- 使用物品：恢复状态、触发效果
- 瞄准：精确攻击弱点

### 3. System（系统）
所属游戏系统

**分类**：
- 移动 (Movement)
- 探索 (Exploration)
- 战斗 (Combat)
- 经济 (Economy)
- 库存 (Inventory)
- 魔法/科技 (Magic/Tech)

### 4. Challenge（挑战）
执行难度的来源

**类型**：
- 时机挑战（精确时机）
- 资源挑战（消耗有限资源）
- 策略挑战（选择最佳方案）
- 反射挑战（快速反应）

### 5. Trigger（触发条件）
何时可以执行该动作？

**示例**：
- 玩家站在可交互物体旁
- 拥有特定物品
- 满足状态条件（满血、特定时间）

### 6. Cue（提示）
如何提示玩家执行时机？

**方式**：
- 视觉提示（高亮、图标）
- 音频提示（音效、语音）
- 触觉反馈（手柄震动）

### 7. Requirements（需求）
需要什么资源或前置条件？

**示例**：
- 需要5个木材
- 需要解锁科技等级2
- 需要特定工具

### 8. Limitations（限制）
频率或效果的限制

**示例**：
- 冷却时间：30秒
- 每日使用次数：3次
- 需要充能时间

### 9. Feedback（反馈）
成功执行的可感知确认

**方式**：
- 视觉（动画、粒子效果）
- 音频（音效、语音）
- UI（数值变化、提示文字）

### 10. Reward（回报）
玩家获得什么？

**类型**：
- 资源（物品、货币）
- 进度（经验、解锁）
- 信息（地图、剧情）
- 情感（成就感、满足感）

## 资源描述八要素

### 模板

```
资源名称：[名称]

用途：
[在游戏中如何使用]

所属系统：
[相关系统]

产生方式：
- [方式1]
- [方式2]

消耗方式：
- [方式1]
- [方式2]

所有权机制：
[如何获得/失去所有权]

存储方式：
[库存、银行、共享仓库等]
```

### 示例：木材

```
资源名称：木材

用途：
- 制作基础工具和武器
- 建造设施
- 作为燃料

所属系统：
- 经济系统
- 制作系统
- 建造系统

产生方式：
- 拆解办公桌椅
- 砍伐（如有外部区域）
- 分解木质收容物残骸

消耗方式：
- 制作物品
- 建造建筑
- 燃烧（供暖/烹饪）

所有权机制：
- 拾取即获得
- 可交易给队友
- 死亡掉落50%

存储方式：
- 个人背包（堆叠上限64）
- 共享仓库（基地存储）
- 物理存在于世界（可临时堆放）
```

## 制作系统设计模式

### 来源
**《游戏中的5种制作系统方法》**  
Tuts+ Code

### 类型4："按需定制" (Made-to-Order Customization)

**特点**：
- 最高复杂度
- 需要玩家更多注意力
- 适合制作是核心元素的游戏

**示例**：
- 《怪物猎人》武器锻造
- 《黑暗之魂》装备升级

### 平衡要点

**核心挑战**：
- 自定义物品 vs 其他途径获得物品的平衡
- 制作投入 vs 产出的价值感
- 复杂度 vs 可理解性

**设计建议**：
- 提供制作预览（投入前可见产出）
- 允许实验性制作（低风险试错）
- 高级制作需要前置知识

## 关键设计模式

### 涌现系统 (Emergent Systems)

**来源**：
**《每个设计师应知的6种游戏设计模式》**

**定义**：
简单规则碰撞产生意外结果

**经典案例**：
- 《我的世界》制作链
- 《旷野之息》物理系统

**应用到生存游戏**：
- 元素交互（火+水=蒸汽）
- 材料属性（金属导电）
- 环境利用（引诱敌人入陷阱）

### 掌握与精通 (Mastery)

**外部奖励 vs 内部满足**：

| 外部奖励 | 内部满足 |
|---------|---------|
| XP/等级 | 技能提升 |
| 装备获取 | 策略优化 |
| 成就解锁 | 心流体验 |

**最佳实践**：
- 两者结合，但内部满足优先
- 技能曲线平滑上升
- 失败提供学习机会

## 生存恐怖资源管理

### 来源
**《资源池的枯竭与恐怖营造》**  
Bjarke the Bard

### Resident Evil效应

**设计模式**：
- 6-8场中等或困难遭遇战组成"冒险日"
- 逐步消耗资源池
- 在资源即将耗尽时提供补给点

### 限制资源池深度

**目的**：
- 防止过早获得过多高级物品
- 保持标准指导线或更低水平
- 维持紧张感

**实现方式**：
- 单次使用物品（如Numenera的Cypher系统）
- 易损装备
- 消耗品占库存空间

### 资源稀缺性真相

**设计幻觉**：
- 资源稀缺是推动玩家谨慎战略的"诡计"
- 实际给予的资源通常"足够+合理误差范围"

**探索奖励**：
- 仔细探索的玩家最终会有盈余
- 鲁莽玩家可能陷入资源危机

## 模板使用建议

### 何时使用完整模板
- 核心机制设计
- 需要团队沟通
- 需要代码实现参考

### 何时简化
- 早期概念阶段
- 个人笔记
- 快速迭代测试

### 迭代流程
1. 概念阶段：一句话描述
2. 原型阶段：简化模板（3-5个要素）
3. 验证阶段：完整模板
4. 优化阶段：根据测试反馈调整
