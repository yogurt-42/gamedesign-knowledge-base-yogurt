# 收容机制游戏化

> 将SCP风格的收容文档转化为可玩的游戏系统

## 核心设计原则

**收容不是背景设定，而是核心玩法。**

> "你不是在'管理'收容物，你是在与它进行一场持续的、充满张力的对话。"

---

## 收容等级游戏化转化

### Safe级：可控资源

**设计定位**：低风险、可预测、可转化为生产力

**游戏机制**：
```
满足度管理 → 稳定产出资源/服务
```

**示例：K-999 "痒痒怪"**
- **满足条件**：定期物理接触（hug）
- **产出**：治疗凝胶（恢复生命值）
- **风险**：满足度低时尝试"粘住"玩家
- **管理策略**：定时维护，防止累积不满

**玩家体验**：
- 安心感（相对安全）
- 资源依赖（形成正循环）
- 疏忽惩罚（提醒玩家责任）

---

### Euclid级：不确定风险

**设计定位**：中等风险、不完全理解、需要主动管理

**游戏机制**：
```
观察学习 → 假设验证 → 调整管理策略 → 应对意外
```

**示例：K-661 "误解之人"**
- **行为模式**：模仿观察者动作
- **复杂性**：延迟时间随暴露缩短
- **风险**：长期暴露后"比原版更好"地执行
- **管理策略**：轮班观察、避免长时间暴露

**玩家体验**：
- 神秘感（不完全理解）
- 学习曲线（逐步掌握规律）
- 紧张感（随时可能失控）

---

### Keter级：生存威胁

**设计定位**：高风险、需要极端措施、随时可能突破

**游戏机制**：
```
多重冗余系统 → 持续监控 → 紧急预案 → 灾难应对
```

**示例：K-001 "哥斯拉原型"**
- **基础需求**：持续高功率供能、多重物理隔离
- **风险因素**：供电波动、外部攻击、内部故障
- **管理策略**：团队配合、预案演练、牺牲决策

**玩家体验**：
- 压迫感（持续的生存压力）
- 团队协作（必须多人配合）
- 道德困境（必要时可能需要牺牲）

## 收容度系统

### 核心参数

```yaml
收容单元状态:
  integrity: 0-100      # 设施完整度
  power_supply: 0-100   # 供电水平
  observation: 0-100    # 观察覆盖度
  satisfaction: 0-100   # 收容物满足度（Safe级）
  activity: 0-100       # 活跃程度（Euclid/Keter）
```

### 动态平衡

**稳态维持**：
```
正常状态：所有参数在安全范围
    ↓
随机波动/玩家行为/外部事件
    ↓
参数偏离 → 预警系统提示
    ↓
玩家干预 → 参数恢复
    ↓
（未及时干预）→ 收容失效
```

### 预警等级

| 颜色 | 等级 | 状态 | 玩家行动 |
|------|------|------|---------|
| 🟢 绿色 | 正常 | 所有参数安全 | 常规维护 |
| 🟡 黄色 | 警告 | 1个参数临界 | 优先处理 |
| 🟠 橙色 | 危险 | 多个参数临界 | 紧急响应 |
| 🔴 红色 | 失效 | 收容突破 | 镇压/撤离 |

## 收容失效动态事件

### 失效触发条件

**渐进式**：
- 参数缓慢恶化（玩家疏忽）
- 最终突破临界点
- 给予预警时间

**突发式**：
- 外部事件（停电、攻击）
- 随机波动（Keter级特性）
- 连锁反应（其他收容物影响）

### 失效事件类型

**类型A：尝试突破**
- 收容物测试收容措施
- 产生可检测的异常迹象
- 玩家有机会强化收容

**类型B：局部失效**
- 收容物部分能力释放
- 影响局部区域
- 需要快速响应但可控

**类型C：完全突破**
- 收容物完全自由
- 全设施警报
- 紧急镇压或撤离

### 失效后果链

```
单个收容物突破
    ↓
警报响起 → 其他收容物活跃度++
    ↓
人员恐慌 → 工作效率下降
    ↓
资源 diverted → 其他收容单元维护减少
    ↓
连锁失效风险
```

## 收容措施作为玩法

### 不是描述，而是操作

**传统SCP文档**：
> "SCP-XXX收容于5m×5m×5m的钢制收容室..."

**游戏化实现**：
- 玩家需要实际建造/维护这个5m³空间
- 钢材需要资源获取
- 尺寸不足会导致收容效果下降
- 材料选择影响收容效果（钢 vs 铅 vs [数据删除]合金）

### 收容设施建造

**模块化设计**：
```
基础收容单元:
├── 隔离层 (材料选择)
├── 观察窗 (单向镜/监控)
├── 交互接口 (安全投喂/采样)
├── 紧急抑制系统 (气体/电击)
└── 自毁装置 (最后手段)
```

**升级路径**：
- Level 1: 基础隔离（Safe级勉强）
- Level 2: 强化结构（Euclid级标准）
- Level 3: 多重冗余（Keter级必需）
- Level 4: 特化改造（针对特定收容物）

## 员工-收容物互动

### 工作类型（脑叶公司启发）

**本能 (Instinct)**：
- 满足生物需求（喂食、清洁）
- 建立基本信任
- 风险：过度接触

**洞察 (Insight)**：
- 观察记录行为
- 发现隐藏模式
- 风险：认知污染（理智下降）

**沟通 (Attachment)**：
- 尝试交流/理解
- 解锁深层信息
- 风险：情感依附、被操控

**压迫 (Repression)**：
- 强制控制/惩罚
- 压制活跃性
- 风险：累积怨恨、报复

### 员工状态影响

**理智值**：
- 高理智：工作效率正常
- 中理智：开始出现幻觉
- 低理智：可能产生错误操作
- 崩溃：危险行为（如故意打开收容单元）

**培训等级**：
- 初级：只能处理Safe级
- 中级：可处理Euclid级
- 高级：可接近Keter级
- 专家：针对特定收容物的特殊训练

## 涌现性事件

### 收容物交互

**示例场景**：
```
K-002 "雷云" 与 K-003 "金属构造体"
    ↓
雷云释放电击 → 金属构造体导电 → 构造体活化
    ↓
联合效应：电气强化构造体，能力增强
    ↓
玩家决策：分离两者 / 利用效应 / 紧急镇压
```

### 实验事故

**有意或无意的实验**：
- 玩家尝试组合两个收容物
- 结果不可预测（可能有益/有害/灾难性）
- 成为新知识或新威胁
