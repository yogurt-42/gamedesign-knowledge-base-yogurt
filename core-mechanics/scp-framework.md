# SCP 风格收容物设计框架

## 核心概念

**不是描述怪物，而是描述如何"处理"非生物因素。**

## 分级系统

### Safe
- **定义**：容易收容，但不一定安全
- **示例**：K-999 痒痒怪，需要定期hug
- **风险**：低，但疏忽会导致逃逸

### Euclid
- **定义**：需要持续观察，行为不完全可预测
- **示例**：K-661 误解之人，会模仿人类
- **风险**：中等，需要主动管理

### Keter
- **定义**：极易突破收容，对世界构成威胁
- **示例**：K-001 哥斯拉原型
- **风险**：极高，需要极端措施

### Thaumiel
- **定义**：用来收容其他的SCP
- **示例**：特殊武器、反制措施
- **用途**：玩家的终极目标资源

## 文档格式模板

```
项目编号：K-XXX

项目等级：[Safe/Euclid/Keter]

特殊收容措施：
[具体如何收容，不是描述怪物本身]
- 房间尺寸
- 电源要求
- 观察频率
- 禁止事项

描述：
[客观描述外观和行为]

附录：
[事故记录、实验日志]
```

## 叙事语调

### 关键特征
- **技术性**：使用科技术语
- **冷漠**：不带情感描述可怕事物
- **官僚**：遵循格式，红acted敏感信息

### 示例对比
**错误**：
> "这是一个可怕的怪物，会吃掉你！"

**正确**：
> "K-661表现出对人类行为的模仿能力。当观察者在██秒内保持静止，项目将重复观察者的上一个动作。长期暴露会导致[数据删除]。"

## 收容物设计原则

### 1. 限制即创意
严格限制催生有趣设计：
- 只能收容在特定材料中
- 需要特定频率的观察
- 对特定刺激有反应

### 2. 反转预期
Safe级可能比Keter级更危险：
- 因为人们掉以轻心
- 因为理解不足

### 3. 故事留白
不要解释一切：
- [数据删除]
- ████
- "原因正在调查中"

### 4. 人性化代价
每个收容物都关联人类故事：
- 前员工的笔记
- 事故中的受害者
- 选择成为实验对象的人

## 给你的游戏：收容物列表模板

### K-999 "痒痒怪" (Safe)
**外观**：橘黄色粘液生物，触感温暖

**收容措施**：
- 室温保持20-25°C
- 每日至少一次物理接触（hug）
- 接触时间不得超过5分钟

**风险**：
- 满足度高：产出治疗凝胶
- 满足度低：尝试逃脱，粘住遇到的任何人

**员工笔记**：
> "第一次接触999时，我哭了。不是害怕，是那种...被理解的感觉。它知道你哪里痛。——研究员陈"

### K-661 "误解之人" (Euclid)
**外观**：人形轮廓，面部模糊

**收容措施**：
- 单向镜面观察室
- 禁止任何模仿行为（包括眨眼）
- 每日更换观察人员

**行为**：
- 完美模仿观察者的动作
- 延迟时间随暴露增加而缩短
- 长期暴露后，项目会"比原版更好"地执行动作

**事故记录**：
> "第47天，它比我先眨眼。我不知道是我模仿它，还是它模仿我。——Dr.██"

## 游戏化机制

### 收容度系统
```
满足度 100% → 产出资源
满足度 50% → 中性状态
满足度 0% → 突破收容
```

### 观察等级
- **Level 1**：外观描述
- **Level 2**：行为模式
- **Level 3**：弱点发现
- **Level 4**：利用方法

升级方式：投入研究时间/资源

### 突破与镇压
**突破条件**：
- 满足度归零
- 外部事件（停电、攻击）
- 随机波动（高等级收容物）

**镇压选项**：
- 物理镇压（伤亡高）
- 利用弱点（需要知识）
- 引诱回收容（风险低但耗时）
